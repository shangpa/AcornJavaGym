# 헬스장 회원 관리 시스템 설계서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | AcornJavaGym - 헬스장 회원 관리 시스템 |
| 언어 | Java |
| 실행 방식 | 콘솔 기반 텍스트 UI |
| 목적 | 헬스장 회원 및 트레이너 관리, 결제, 출석, 운동 기구 안내 기능 제공 |

---

## 2. 디렉토리 구조

```
Gym/src/
├── Main.java                          # 진입점 및 메인 메뉴
├── interfaces/
│   ├── DietManageable.java            # 식단 관리 인터페이스
│   ├── FitnessEquipmentGuide.java     # 운동 기구 안내 인터페이스
│   ├── Payable.java                   # 결제 인터페이스
│   └── Registerable.java             # 등록 인터페이스
├── manager/
│   └── GymManager.java               # 회원/트레이너 통합 관리자
└── model/
    ├── Person.java                    # 추상 기반 클래스
    ├── Member.java                    # 회원 클래스
    ├── Trainer.java                   # 트레이너 클래스
    ├── MembershipType.java            # 회원권 종류
    └── TrainerGrade.java              # 트레이너 등급 (Enum)
```

---

## 3. 클래스 상세 설명

### 3.1 인터페이스

#### `Registerable`
- **위치**: `interfaces/Registerable.java`
- **역할**: 등록 기능 계약 정의
- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `register()` | void | 등록 처리 |

#### `Payable`
- **위치**: `interfaces/Payable.java`
- **역할**: 결제 기능 계약 정의
- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `pay(int months)` | void | 개월 수 기반 결제 처리 |

#### `DietManageable`
- **위치**: `interfaces/DietManageable.java`
- **역할**: 식단 관리 기능 계약 정의
- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `manageDiet(Member member)` | void | 회원 식단 관리 |

#### `FitnessEquipmentGuide`
- **위치**: `interfaces/FitnessEquipmentGuide.java`
- **역할**: 운동 기구 안내 계약 정의
- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `explainEquipment(String bodyPart)` | void | 부위별 운동 기구 안내 |

---

### 3.2 열거형 (Enum)

#### `TrainerGrade`
- **위치**: `model/TrainerGrade.java`
- **역할**: 트레이너 등급 정의 (담당 회원 수 기준 자동 승급)
- **상수**:
  | 상수 | 담당 회원 수 | 설명 |
  |------|------------|------|
  | `INTERN` | 0 ~ 3명 | 인턴 |
  | `JUNIOR` | 4 ~ 6명 | 주니어 |
  | `SENIOR` | 7 ~ 9명 | 시니어 |
  | `MASTER` | 10명 이상 | 마스터 |

---

### 3.3 모델 클래스

#### `Person` (추상 클래스)
- **위치**: `model/Person.java`
- **역할**: 회원과 트레이너의 공통 기반 클래스
- **필드**:
  | 필드명 | 타입 | 설명 |
  |--------|------|------|
  | `name` | String | 이름 |
  | `phoneNumber` | String | 전화번호 |
  | `age` | int | 나이 |
  | `height` | double | 신장 (cm) |
  | `weight` | double | 체중 (kg) |

- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `getBMI(double height, double weight)` | double | BMI 계산 |
  | `getBMICategory()` | void | BMI 범주 출력 (저체중/정상/과체중/비만) |
  | `printInfo()` | void (abstract) | 정보 출력 (서브클래스 구현) |
  | Getter/Setter | - | 각 필드 접근자 |

---

#### `Member`
- **위치**: `model/Member.java`
- **역할**: 헬스장 회원 정보 및 기능 관리
- **상속**: `Person`
- **구현 인터페이스**: `Registerable`, `Payable`, `FitnessEquipmentGuide`
- **필드**:
  | 필드명 | 타입 | 설명 |
  |--------|------|------|
  | `registrationDate` | LocalDate | 등록일 |
  | `expirationDate` | LocalDate | 만료일 |
  | `expirationDate_after_extension` | LocalDate | 연장 후 만료일 |
  | `attendedToday` | boolean | 오늘 출석 여부 |
  | `membership` | MembershipType | 선택한 회원권 |
  | `formatter` | DateTimeFormatter | 날짜 포맷 (yyyy년 MM월 dd일) |

- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `register()` | void | 오늘 날짜로 등록일 설정 |
  | `pay(int months)` | void | 결제 및 만료일 갱신 |
  | `explainEquipment(String bodyPart)` | void | 상체/하체/유산소 기구 안내 |
  | `checkIn()` | void | 오늘 출석 처리 |
  | `printInfo()` | void | 회원 정보 출력 |

---

#### `Trainer`
- **위치**: `model/Trainer.java`
- **역할**: 트레이너 정보 및 회원 관리 기능
- **상속**: `Person`
- **구현 인터페이스**: `Registerable`, `DietManageable`
- **필드**:
  | 필드명 | 타입 | 설명 |
  |--------|------|------|
  | `grade` | TrainerGrade | 현재 등급 (기본값: INTERN) |
  | `managedMembers` | ArrayList\<Member\> | 담당 회원 목록 |

- **메서드**:
  | 메서드 | 반환 타입 | 설명 |
  |--------|-----------|------|
  | `register()` | void | 트레이너 등록 및 등급 출력 |
  | `manageDiet(Member member)` | void | 회원 BMI 기반 식단 추천 |
  | `addManagedMember(Member member)` | void | 담당 회원 추가 및 등급 자동 갱신 |
  | `updateGradeByMemberCount()` | void | 회원 수 기반 등급 업데이트 (private) |
  | `printAttendance()` | void | 오늘 출석한 담당 회원 출력 |
  | `printInfo()` | void | 트레이너 정보 전체 출력 |

---

#### `MembershipType`
- **위치**: `model/MembershipType.java`
- **역할**: 회원권 종류 및 가격 정의
- **정적 상수 (미리 정의된 플랜)**:
  | 상수 | 기간 | 가격 |
  |------|------|------|
  | `ONE_MONTH` | 1개월 | 100,000원 |
  | `THREE_MONTH` | 3개월 | 270,000원 |
  | `SIX_MONTH` | 6개월 | 500,000원 |
  | `TWELVE_MONTHS` | 12개월 | 900,000원 |

- **필드**:
  | 필드명 | 타입 | 설명 |
  |--------|------|------|
  | `months` | final int | 이용 개월 수 |
  | `price` | final int | 가격 (원) |

## 4. 메뉴 구조

```
[메인 메뉴]
├── 1. 회원 메뉴
│   ├── 1. 회원 등록 (이름, 전화번호, 나이, 신장, 체중 입력)
│   │       └── 회원권 선택 (1/3/6/12개월)
│   ├── 2. 결제 (추가 결제)
│   └── 3. 출석 체크
│
├── 2. 트레이너 메뉴
│   ├── 1. 트레이너 등록
│   ├── 2. 담당 회원 추가 (자동 등급 승급)
│   ├── 3. 트레이너 정보 조회
│   ├── 4. 담당 회원 식단 관리 (BMI 기반)
│   └── 5. 오늘 출석 확인
│
└── 3. 운동 기구 안내 메뉴
    ├── 1. 상체 운동 기구
    ├── 2. 하체 운동 기구
    └── 3. 유산소 기구
```

## 5. 핵심 설계 원칙

### OOP 원칙 적용
| 원칙 | 적용 내용 |
|------|-----------|
| **상속 (Inheritance)** | `Person` → `Member`, `Trainer` 계층 구조 |
| **캡슐화 (Encapsulation)** | private 필드 + public getter/setter |
| **다형성 (Polymorphism)** | `printInfo()` 추상 메서드 오버라이딩 |
| **추상화 (Abstraction)** | 4개 인터페이스로 기능 계약 분리 |

### 의존성 요약
| 클래스 | 의존 대상 |
|--------|-----------|
| Main | GymManager, Member, Trainer, MembershipType |
| GymManager | Member, Trainer |
| Member | Person, MembershipType |
| Trainer | Person, Member, TrainerGrade |
| Person | (없음 - 기반 클래스) |
| MembershipType | (없음) |
| TrainerGrade | (없음 - Enum) |
